# Documentation

This directory contains all project documentation for the EV Controller system.

## Contents

### Core Documents

- **`requirements.md`** - System requirements specification including functional and non-functional requirements
- **`design-specifications.md`** - Detailed design documentation for all subsystems
- **`project-proposal.pdf`** - Initial project proposal (to be added)

### Presentations

The `presentations/` folder contains:
- Project presentations
- Progress reports
- Final presentation materials

## Document Organization

### Requirements Documentation
The requirements document (`requirements.md`) defines:
- Functional requirements (FR-xxx-yyy)
- Non-functional requirements (NFR-xxx-yyy)
- Safety and performance specifications
- Test and validation criteria

### Design Documentation
The design specifications document (`design-specifications.md`) includes:
- System architecture
- Subsystem designs (Motor Control, Cruise Control, ACC)
- Control algorithms and parameters
- Interface specifications
- Simulink model structure

## Contributing to Documentation

When adding or updating documentation:

1. Follow the established naming convention
2. Use clear section headings and numbering
3. Include version control information
4. Update this README when adding new documents
5. Add relevant cross-references between documents

## Document Formats

- **Markdown (.md)** - For living documents that change frequently
- **PDF (.pdf)** - For formal submissions and presentations
- **PowerPoint (.pptx)** - For presentations

## Version Control

All documents should include:
- Version number
- Date of last update
- Author/team member responsible
- Change summary

## Review Process

1. Draft documents in the appropriate format
2. Share with team for review
3. Incorporate feedback
4. Get approval from project lead
5. Archive old versions if necessary
